LSTM Based Ensemble Network To Enhance The Learning Of Long-Term Dependencies In Chatbot

ABSTRACT:

With the advent of Artificial Intelligence, chatbots have evolved from being mere rule-based models to more sophisticated models which are not only inclined towards the humane aspect of basic conversations but also tend to mature over experiences. One of the most striking features of the existing chatbot systems is their ability to maintain and nurture specific features and contexts of the conversations enabling them to have a humanly touch throughout the experience. The project proposed a hybrid Long Short Term Memory based Ensemble Network classification model to preserve the continuance of the context. The proposed model utilizes a defined number of Long Short Term Memory Networks as part of the assembled model working in unison to generate the collective prediction class for the input query processed.

Basic Ensemble Structure

METHODOLOGY:

The idea is to define a number of LSTM networks with variation in hyperparameters as part of the ensemble model. The member models work together in parallel and their individual outputs are aggregated to generate the output of the overall model. The primary benefit of the Ensemble Model comes in the form of better generalization ability. The proposed model includes Training & Testing phases.
  1. Training Phase:
    2. A definite number of LSTM networks are generated and trained using variations in training data. The models with lower accuracy are filtered out.
    • a. Parameters like number of LSTM network, Maximum number of hidden layers, the maximum number of neurons and accuracy threshold for each LSTM model are initialized.
    • b. LSTM models are generated.
    • c. The training dataset is split and the training is performed for LSTM models. A portion of dataset is used for testing and the LSTM models having lower accuracy than the threshold are dropped.
    Variations in LSTM
  2. Testing Phase:
    3. The trained model is applied to the testing dataset.
    • a. The testing dataset is fed into LSTM models.
    • b. Individual outputs from the model are multiplied with corresponding weights and a weighted average of the sum of total outputs is calculated.
    • c. The weighted average is used to predict the response class.

DATASET USED:

Cornell Movie--Dialogs Corpus The dataset contains 220,579 conversational exchanges between 10,292 pairs of movie characters. It involves 9,035 characters from 617 movies. All the files have field separator as " +++$+++ ". The project utilizes “movie_lines.txt” and “movie_conversations.txt” for training the model. ‘movie_conversations.txt’ contains the structure of conversations with the characterIDs of the characters involved in the conversations. ‘movie_lines.txt‘ contains the actual texts of the conversations with lineID and the characterIDs.

PROGRAMMATIC APPROACH:

  1. Data Preprocessing
    2. The dataset is imported and corresponding dictionaries are created mapping line in conversation to its identifier. Dictionaries for questions and answers are also created. The questions and answers are cleaned to simplify the tokens.
  2. Building seq2seq model
    3. The placeholders for model inputs are defined. Encoder RNN layer with parameters like Model Inputs, Input questions, targets are defined. Decoder RNN for carrying out the decoding the training dataset, decoding validation dataset is defined.
  3. Training the Seq2Seq Model
    4. The Hyperparameters like epochs, batch size, Layers, Learning rate, etc are initialized and The dataset is split into batches for training. After the training starts for each iteration, the Weights generated are stored as ‘chatbot_weights.ckpt’ checkpoint file.
    The model during training
  4. Testing
    5. The weights file is loaded for a session. The questions are preprocessed and the Seq2Seq model is called for execution. The response generated is returned for the input query.

GRU OR LSTM?

The selection of an appropriate model between them depends on the key differences and the dataset. GRU exposes complete memory content without control gate when compared to controlled exposure of LSTM using Output gate. LSTM doesn't control the amount of information flowing in from previous time steps while computing new memory content. On the other hand, GRU explicitly controls the influx of information while calculating new memory content using the Update gate. Comparison between LSTM and GRU for the model appropriate model selection is done over the dataset in 3 iterations. It was found that as the data points in the dataset were increased the performance of LSTM was better than GRU.

CONCLUSION

Chatbot console output
The project utilized the concept of Ensemble learning over the traditional approach of employing a single methodology to carry out the task of classification for answers in Chatbot. In addition to creating an Ensemble, the project focused on the concept of pruning as well to refine the individual LSTM models involved to improve the overall accuracy of the model. The project also compares the selection between LSTM and GRU as the element of memory for the Ensemble Network. In comparison, LSTM was found well suited for classification dealing with a large volume of the training dataset.

The proposed LSTM based Ensemble Network architecture attempts at enhancing the user experience by providing a sense of continuance of context in a series of conversations. The algorithm does so by generalizing the features imperative to making the conversation humane.
